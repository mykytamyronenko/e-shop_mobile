# e-Shop Mobile

Application mobile développée dans le cadre de mes études, correspondant à la version mobile frontend du projet **e-Shop**.

Cette application consomme la **même API REST** que la version web du projet, permettant de partager la logique métier et les données entre plusieurs clients (web et mobile).

## Contexte

- Projet scolaire
- Réalisé en complément de la version web e-Shop
- Objectif : démontrer l’utilisation d’une API REST depuis une application mobile Android

## Fonctionnalités

- Consultation du catalogue de produits (images incluses)
- Authentification des utilisateurs
- Achat de produits pour les utilisateurs connectés
- Communication avec une API REST

## Technologies utilisées

- **Langage** : Kotlin
- **Plateforme** : Android
- **Architecture** : Client mobile consommant une API REST
- **Communication réseau** : HTTP / JSON

## Ce que ce projet démontre

- Consommation d’une API REST depuis une application mobile
- Partage d’un backend entre plusieurs clients
- Compréhension de l’architecture client / serveur
- Développement mobile avec Kotlin
